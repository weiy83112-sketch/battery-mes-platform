## 今天阅读的代码

function createWorkOrder(event: FormEvent) {
  event.preventDefault();
  void submit(
    "/work-orders",
    {
      product_name: productName.trim(),
      planned_quantity: Number(plannedQuantity),
      status: "待生产",
    },

## 已建立的理解

### 1. `WorkOrder` 为什么要定义类型

当前理解：

```text
WorkOrder 用来描述一条工单数据应该有哪些字段，以及每个字段是什么类型。
```

TypeScript 可以在开发和编译阶段帮助检查：

- 是否缺少字段
- 字段名称是否正确
- 字段类型是否正确

需要纠正：它不只是“限制参数”，也不会在浏览器运行时自动校验后端数据。

### 2. 输入框内容保存在哪里

输入框内容先保存在 React 状态中：

```tsx
const [productName, setProductName] = useState("");
const [plannedQuantity, setPlannedQuantity] = useState("");
```

需要纠正：输入内容不是直接保存在 JSON 中。

完整过程是：

```text
用户输入
→ React 状态保存字符串
→ 提交时整理成 JavaScript 对象
→ JSON.stringify 转换成 JSON 字符串
→ 发送给后端
```

### 3. 点击提交后执行什么

点击提交后，首先执行前端事件函数：

```tsx
createWorkOrder(event)
```

然后调用：

```tsx
submit(...)
```

请求顺序是：

```text
点击提交
→ createWorkOrder
→ submit
→ POST /work-orders 创建工单
→ 创建成功
→ refresh
→ GET /work-orders 获取最新工单列表
→ React 更新页面
```

因此不是在 GET 和 POST 中二选一，而是先使用 POST 创建，再使用 GET 刷新列表。

## 今天接触的语法

```text
function                   声明函数
event: FormEvent           给参数标注 React 表单事件类型
event.preventDefault()     阻止表单默认刷新页面
{ key: value }             创建 JavaScript 对象
productName.trim()         删除字符串两端空格
Number(value)              把字符串转换成数字
() => { ... }              创建箭头回调函数
void submit(...)           调用异步函数，但不使用返回的 Promise
```
