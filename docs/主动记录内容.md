### Python 返回值类型提示：`list[dict[str, int | str]]`

示例：

```python
def list_work_orders() -> list[dict[str, int | str]]:
    return work_orders
```

其中：

```python
-> list[dict[str, int | str]]
```

叫返回值类型提示。它不是业务代码，而是在说明这个函数预计返回什么类型的数据。

#### 对应的数据

```python
[
    {
        "id": 1001,
        "product_name": "动力电池模组",
        "planned_quantity": 500,
        "status": "生产中",
    }
]
```

#### 一层一层理解

最外层是方括号：

```python
[
  ...
]
```

所以第一层是列表：

```python
list
```

列表里的每一项是花括号：

```python
{
    "id": 1001,
    "product_name": "动力电池模组",
}
```

所以列表中的每一项是字典：

```python
dict
```

字典里的 key：

```python
"id"
"product_name"
"planned_quantity"
"status"
```

都是字符串，所以 key 的类型是：

```python
str
```

字典里的 value 有整数：

```python
1001
500
```

也有字符串：

```python
"动力电池模组"
"生产中"
```

所以 value 的类型是：

```python
int | str
```

`|` 表示“或者”。

#### 合起来读

```python
list[dict[str, int | str]]
```

人话：

```text
返回一个列表。
列表里每一项是字典。
字典的 key 是字符串。
字典的 value 是整数或字符串。
```

对应关系：

```text
最外层 []                     → list
里面每一个 {}                 → dict
"id"、"product_name"          → str 类型的 key
1001、500                     → int 类型的 value
"动力电池模组"、"生产中"       → str 类型的 value
```

#### 需要记住

```text
类型提示像数据说明书。
它不会创建数据，只是说明函数预计返回什么结构。
```

初学阶段常见写法：

```python
-> dict[str, str]
```

返回一个字典，key 和 value 都是字符串。

```python
-> dict[str, int]
```

返回一个字典，key 是字符串，value 是整数。

```python
-> list[dict[str, int | str]]
```

返回一个列表，里面每一项是字典，字典里既有整数值，也有字符串值。

## React 列表渲染

### 什么时候使用

当页面需要展示多条结构相同的数据时使用，例如：

- 设备列表
- 工单列表
- 报警列表
- 条码过站记录

不要为每条数据手写一个组件。先把数据放进数组，再使用 `map` 生成组件。

### 标准写法

```tsx
const devices = [
  { id: 1001, name: "焊接机", status: "运行中" },
  { id: 1002, name: "测试机", status: "待机" },
  { id: 1003, name: "装配机", status: "异常" },
];

{devices.map((device) => (
  <DeviceStatus
    key={device.id}
    deviceId={device.id}
    name={device.name}
    status={device.status}
  />
))}
```

### 怎么理解和使用

```text
devices.map(...)
```

表示依次读取 `devices` 数组中的每一条数据，并为每条数据生成一个 `DeviceStatus` 组件。

```tsx
device.id
device.name
device.status
```

表示读取当前设备的编号、名称和状态。

```tsx
key={device.id}
```

列表中的每个组件都需要一个稳定且唯一的 `key`，通常使用数据库记录的 `id`。

`key` 用于帮助 React 区分列表项目，不会作为普通 Props 传给子组件。

### 需要记住

```text
数组保存多条数据
map 把每条数据转换成一个组件
key 使用稳定且唯一的 id
通过 Props 把当前数据传给子组件
```

### MES 中的常见应用

后端返回设备列表：

```json
[
  {"id": 1001, "name": "焊接机", "status": "运行中"},
  {"id": 1002, "name": "测试机", "status": "待机"}
]
```

前端接收数组后，使用 `map` 将每台设备显示为一个设备状态组件。

## 前端调用后端健康检查接口

### 功能目标

页面打开后，前端自动请求 FastAPI 的 `/health` 接口，并展示后端服务状态：

```text
检查中 → 在线
检查中 → 离线
```

### 完整执行流程

```text
用户打开前端页面
→ 页面先显示“后端服务：检查中”
→ useEffect 调用健康检查函数
→ fetch 请求 FastAPI 的 GET /health
→ 后端返回 {"status": "ok"}
→ setBackendStatus 将状态改为“在线”
→ React 更新页面
```

如果请求失败，则把状态修改为“离线”。

### 客户端组件

```tsx
"use client";
```

需要在浏览器中使用 `useState`、`useEffect`、点击事件等 React 交互功能时，在文件顶部添加 `"use client"`。

### 保存和修改页面状态

```tsx
type BackendStatus = "检查中" | "在线" | "离线";

const [backendStatus, setBackendStatus] =
  useState<BackendStatus>("检查中");
```

需要记住：

```text
backendStatus：读取当前状态
setBackendStatus：修改状态并让 React 更新页面
"检查中"：初始状态
```

自己的理解：

```text
useState 先把初始值交给状态数据名。
页面中读取这个数据名来显示内容。
当调用 set数据名(...) 时，React 会更新状态。
状态更新后，React 会重新渲染相关组件，页面就会显示新结果。
```

例如：

```tsx
const [backendStatus, setBackendStatus] = useState("检查中");
```

可以理解为：

```text
backendStatus 初始值是“检查中”
调用 setBackendStatus("在线")
React 把 backendStatus 更新为“在线”
页面重新显示最新状态
```

修改状态：

```tsx
setBackendStatus("在线");
```

不要直接修改：

```tsx
backendStatus = "在线";
```

### 页面打开后执行一次

```tsx
useEffect(() => {
  checkBackendHealth();
}, []);
```

这里的空数组 `[]` 表示页面首次打开后执行一次。

### 请求后端并读取 JSON

```tsx
async function checkBackendHealth() {
  try {
    const response = await fetch("http://127.0.0.1:8000/health");

    if (!response.ok) {
      throw new Error("后端健康检查失败");
    }

    const data: { status: string } = await response.json();
    setBackendStatus(data.status === "ok" ? "在线" : "离线");
  } catch {
    setBackendStatus("离线");
  }
}
```

需要记住：

```text
async：函数中需要等待异步操作
await fetch(...)：发送请求并等待 HTTP 响应
response.ok：判断 HTTP 请求是否成功
await response.json()：读取后端返回的 JSON
try / catch：请求失败时执行备用处理
```

### 判断的简写方式

```tsx
setBackendStatus(data.status === "ok" ? "在线" : "离线");
```

相当于：

```tsx
if (data.status === "ok") {
  setBackendStatus("在线");
} else {
  setBackendStatus("离线");
}
```

### CORS 跨域配置

前端和后端地址不同：

```text
前端：http://localhost:3000
后端：http://127.0.0.1:8000
```

浏览器默认会限制前端跨来源访问后端，因此 FastAPI 需要明确允许前端来源：

```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],
    allow_methods=["*"],
    allow_headers=["*"],
)
```

可以理解为后端告诉浏览器：

```text
允许 http://localhost:3000 访问我的接口
```

## FastAPI GET 列表接口

### 功能目标

返回多条结构相同的数据，例如工单列表、设备列表、报警列表。

### 示例代码

```python
work_orders = [
    {
        "id": 1001,
        "product_name": "动力电池模组",
        "planned_quantity": 500,
        "status": "生产中",
    },
    {
        "id": 1002,
        "product_name": "储能电池包",
        "planned_quantity": 300,
        "status": "待生产",
    },
]


@app.get("/work-orders")
def list_work_orders() -> list[dict[str, int | str]]:
    """返回模拟工单列表。"""
    return work_orders
```

### 怎么理解

```text
work_orders：临时模拟数据列表
@app.get("/work-orders")：注册 GET 工单列表接口
list_work_orders：接口对应的处理函数
return work_orders：把工单列表返回给前端
```

### 返回值类型

```python
list[dict[str, int | str]]
```

表示：

```text
返回的是一个列表
列表中的每一项是一个字典
字典的 key 是字符串
字典的 value 可能是整数或字符串
```

### MES 业务含义

工单是生产任务。例如：

```text
生产什么产品
计划生产多少
当前处于什么状态
```

真实系统中，工单列表最终会来自数据库。当前阶段先用模拟数据学习接口结构。
