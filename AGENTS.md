# AGENTS.md

## 这个文件的作用

这个文件用来约定本项目的协作方式，方便在后续开发中保持统一节奏。

## 当前协作约定

1. 每次只做一个小步骤。
2. 每次修改前，先说明要改哪些文件。
3. 每执行一个命令，都解释命令的作用。
4. 每完成一个功能，都说明如何运行和测试。
5. 尽量使用简单、容易理解的代码。
6. 代码里尽量补充中文注释，帮助学习。
7. 不随意改动无关代码。
8. 当前阶段优先学习项目结构、基础工程化和核心业务建模。

## 目录职责约定

- `backend/`：后端 API、数据库模型、业务逻辑
- `frontend/`：前端页面、组件、接口调用
- `docs/`：文档、设计说明、学习记录

## 后续建议

后面我们可以继续按下面顺序推进：

1. 初始化 Git 仓库
2. 初始化前端 `Next.js` 项目
3. 初始化后端 `FastAPI` 项目
4. 跑通前后端最小示例
5. 再开始进入工单、条码、追溯等业务模块
